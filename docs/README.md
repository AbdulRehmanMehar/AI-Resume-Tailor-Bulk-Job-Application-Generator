# Documentation Directory

This directory contains all project documentation, architectural decisions, and implementation guides for the Resume Tailor application.

## Contents

### Implementation Guides

- **`COMPLETE_CONDITIONAL_SYSTEM.md`**: Complete guide to the conditional field system
- **`COMPREHENSIVE_DOCX_SOLUTION.md`**: DOCX text extraction implementation
- **`CONDITIONAL_CONTACT_SYSTEM_COMPLETE.md`**: Contact information preservation system
- **`DOCX_EXTRACTION_SOLUTION.md`**: Document processing solutions
- **`ENHANCED_SCHEMA_COMPLETE.md`**: API schema enhancements
- **`INTEGRATION_COMPLETE.md`**: System integration documentation

### Summary Scripts

- **`comprehensive-docx-solution.js`**: Overview of DOCX extraction capabilities
- **`docx-solution-summary.js`**: Summary of document processing solutions
- **`enhanced-schema-overview.js`**: Schema enhancement feature overview

## Key Features Documented

### Conditional Field System

- Section-level conditional rendering (`has_professional_summary`, `has_skills`, etc.)
- Sub-field conditional rendering within objects (`has_job_title`, `has_company`, etc.)
- Contact information preservation and validation
- Data integrity and fabrication prevention

### Document Processing

- Comprehensive DOCX text extraction
- Header, footer, and embedded content extraction
- Formatted text preservation
- Table and list content extraction

### API Architecture

- OpenAI function calling schema
- Structured data validation
- Error handling and response formatting
- Backward compatibility maintenance

## Usage

Documentation files use Markdown format and can be viewed in any Markdown reader or directly on GitHub.

Summary scripts can be executed with Node.js:

```bash
node docs/enhanced-schema-overview.js
```
